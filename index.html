Dis Managing the Underworld of Database Testing

A unit testing framework for PostgreSQL Databases that focuses on simplicity, robustness, manageability, and reporting

Download v0.4 View on GitHub

Rollback Guarantee

All properly constructed tests will return an exception which will rollback changes made during the test. Additionally, the test runners wrap the execution of a test in a function that throws an exception to ensure rollback even if the test is improperly constructed. This also ensures that each test is sandboxed from the other tests.

Test Execution

With Dis you no longer need to maintain scripts for executing tests. The Dis test runner will execute all tests or limted to schema, module, submodule, or individual test.

Detailed Reporting

Dis reports on the result of every assertion tested with detailed information on any failure. Dis also maintains a series of views and functions for both summary and detailed reporting on the most recent run of tests at the global, schema, module, submodule, test, and assertion level. Results are stored in the database rather than relying on some external process.

Unit Tested

Dis is rigorously tested, and since all tests are included you can continue to verify its validity.

Database Exclusive

All operations are performed within the database so there are no external dependencies.
